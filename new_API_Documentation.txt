AUTHENTICATION
All API requests require authentication using the X-API-Key header. Your API key is unique to your account and
should be kept confidential.
AUTHENTICATION HEADER
X-API-Key: api_817411c42ccea3cfe8dfdf9c68af682733c8febb70331775fddf9a6a32af56bd

SECURITY CHECKLIST
•

Never expose keys in client-side code

•

Store keys in environment variables

•

Enable domain restrictions in settings

•

Rotate keys if compromised

POST

/api/purchase

Purchase Data
Purchase data bundles for recipients

PARAMETERS
networkKey

string

Network identifier

recipient

string

Phone number (10 digits)

capacity

number

Data in GB

EXAMPLE
curl -X POST "https://encartastores.com/api/purchase" \
-H "X-API-Key: your_key" \
-H "Content-Type: application/json" \
-d '{"networkKey":"YELLO","recipient":"0241234567","capacity":1}'

RATE LIMITING
Fair Usage
Maximum 100 requests per minute per API key. Exceeding limits returns HTTP 429.

RESPONSE HEADERS
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1706198400
